  Утилита предназначена для обновления ПО ФР. 
  
  Алгоритм работы утилиты:
  1. Получение информации о ККМ
  2. Чтение денежного регистра, если RestoreCashRegister = true
  3. Чтение значений таблиц, если SaveTables = true
  4. Проверка, что состояние ККМ = 4, закрытая смена
  5. Форматирование SD карты
  6. Поиск в цикле подходящего загрузчика и запись его через DFU или XModem
     Таким образом можно записать несколько загрузчиков, например: 
	 153 -> 155 -> 1939
  7. Поиск и обновление ПО ФР через DFU или XModem
	 После обновления ПО ФР таблицы принимают значения по-умолчанию.
     Поэтому если подключение было TCP через RNDIS, то оно может стать VCOM.
	 Поиск устройства и проверка версии ПО ФР.
  8. Если параметр SaveTables = true, то записываем значения таблиц
  9. Если для ПО ФР заданы дополнительные значения таблиц, то они записываются.
  10. Если в файле rules.json есть элемент записи таблиц, ACTION_WRITE_TABLES = 6,
      то записываем эти таблицы
  11. Перезагружаем ФР и повторно читаем параметры драйвера ФР, метод LoadParams.
      Драйвер читает параметры как при загрузке. Проверяем связь с ФР. 
  12. Выполняем действия инициализации архива ФН, фискализации, перефискализации или записи лицензии.

*******************************************************************

  Раздел «ФН сервис» — вкладка «Архив ФН»:

  Вкладка предназначена для работы с архивом фискального накопителя (ФН)
  и содержит три функциональных блока:

  1. Выгрузка архива ФН (для ЦГРП)
     - Номер первого ФД  — порядковый номер первого фискального документа
       в диапазоне выгрузки.
     - Номер последнего ФД — порядковый номер последнего ФД в диапазоне.
       Кнопка «Прочитать номер последнего ФД» позволяет получить это
       значение автоматически из ФН.
     - Флаг «Включить только чеки с тегом 1162» — при установке в архив
       попадают только документы, содержащие тег 1162 (маркированный товар).
     - Кнопка «Прочитать и сохранить архив ФН в файл...» — считывает
       фискальные документы из ФН в заданном диапазоне и сохраняет их
       в бинарный файл архива.

  2. Отчёт о считывании ФД для ФФД 1.2 (файлы .fnc и .txt)
     - Кнопка «Сформировать» — запускает формирование отчёта о считывании
       фискальных документов в форматах .fnc и .txt (ФФД 1.2).
     - Индикатор прогресса — отображает ход выполнения операции (0–100 %).
     - Кнопка «Прервать» — прерывает выполняемую операцию формирования отчёта.

  3. Отчёт о реализации маркированного товара (ФФД 1.2)
     - Флаг «Выгрузить все уведомления» — при установке выгружаются все
       уведомления о реализации маркированного товара; иначе выгружается
       диапазон, начиная с указанного номера.
     - Номер первого увед. — номер первого уведомления для выгрузки.
     - Кол-во уведомлений — количество уведомлений, доступных в ФН
       (заполняется после нажатия кнопки «Запросить кол-во уведомлений»).
     - Кнопка «Запросить кол-во уведомлений» — запрашивает у ФН общее
       количество уведомлений о реализации маркированного товара.
     - Кнопка «Выгрузить уведомления в файл...» — сохраняет уведомления
       в файл на диске.

  ACTION_UPDATE_FIRMWARE  = 2; // Обновление программы

  ACTION_INIT_FS          = 4; // Инициализация тестового ФН
  ACTION_FISCALIZE_FS     = 5; // Фискализация ФН
  ACTION_WRITE_TABLES     = 6; // Запись таблиц
  ACTION_REFISCALIZE_FS   = 7; // Перерегистрация ФН
